## ADM-01 · 관리자 홈

**진입 경로:** 로그인 성공 (관리자) / 바텀 네비 > 홈 탭

---

### 레이아웃

```
┌─────────────────────────────┐
│  아차사고 관리          🔔 ☰ │  ← 상단바
├─────────────────────────────┤
│                             │
│  ┌─────────────────────┐   │
│  │  부대 신고 현황      │   │  ← 신고 현황 요약 카드
│  │                     │   │
│  │  전체   조치완료  대기│   │
│  │   47     38      9  │   │
│  └─────────────────────┘   │
│                             │
│  미조치 건 현황              │  ← 섹션 라벨
│                             │
│  ┌─────────────────────┐   │
│  │ 조치 대기     9건   │   │  ← 미조치 현황 카드
│  │ ⚠ 기한 초과   3건   │   │  (기한 초과 강조)
│  │              [확인] │   │
│  └─────────────────────┘   │
│                             │
│  빠른 메뉴                   │  ← 섹션 라벨
│                             │
│  ┌──────┐ ┌──────┐ ┌──────┐│
│  │  📋  │ │  📊  │ │  👤  ││
│  │사고  │ │신고  │ │계정  ││
│  │등록  │ │현황  │ │승인  ││
│  └──────┘ └──────┘ └──────┘│
│                             │
└─────────────────────────────┘
│  🏠홈  📋등록  📊현황  📢공지  👤내정보 │  ← 바텀 네비
└─────────────────────────────┘
```

---

### 컴포넌트 상세

#### 상단바
- 좌: 앱 타이틀 "아차사고 관리"
- 우: 벨 아이콘 (🔔) — 읽지 않은 알림 시 숫자 뱃지
- 우: 햄버거 메뉴 (☰) — 탭 시 통계 그래프 화면으로 이동

#### 신고 현황 요약 카드
- 전체 등록 건수 / 조치 완료 건수 / 조치 대기 건수 (3열)
- 조치 대기: 강조 색상 (주황)
- 카드 탭: ADM-02 (전체 신고 현황)으로 이동
- 집계 범위: 관리자 소속 부대 전체

#### 미조치 건 현황 카드
- 조치 대기 건수 (전체)
- 기한 초과 건수 — 강조 색상 (빨강), 경고 아이콘 (⚠)
- [확인] 버튼: ADM-03 (미조치 추적 관리)으로 이동
- 기한 초과 0건: 경고 표시 없이 "조치 대기 N건" 만 표시

#### 빠른 메뉴
- [사고 등록]: REG-01로 이동
- [신고 현황]: ADM-02로 이동
- [계정 승인]: ADM-04로 이동
- 3열 그리드, 아이콘 + 텍스트 라벨
- 미승인 대기 건 있을 시 [계정 승인] 아이콘에 숫자 뱃지 표시

#### 바텀 네비게이션
- 홈 / 사고 등록 / 신고 현황 / 공지사항 / 내 정보 (5탭)
- 현재 탭 강조

---

### 상태별 UI

| 상태 | UI 처리 |
|---|---|
| 기한 초과 건 있음 | 미조치 카드 빨강 강조 + 경고 아이콘 |
| 미승인 계정 대기 있음 | 빠른 메뉴 계정 승인 뱃지 |
| 읽지 않은 알림 있음 | 벨 아이콘 숫자 뱃지 |

### 컴포넌트 상세

#### 등록자 필터 탭
- [내가 등록한 건] / [부대 전체 등록 건] 2탭
- 전환 시 목록 즉시 갱신

#### 필터 칩 바 (가로 스크롤)
- [필터] 전체 열기 버튼 — 탭 시 필터 선택 바텀시트
- [사고유형] 칩 — 탭 시 유형 선택 바텀시트
- [처리상태] 칩 — 탭 시 조치 대기 / 조치 완료 선택
- [기간] 칩 — 탭 시 날짜 범위 선택
- 필터 적용 시 칩 강조 색상으로 변경
- [초기화] 버튼 — 필터 적용 상태에서만 표시

#### 필터 선택 바텀시트
```
┌─────────────────────────────┐
│  필터              [초기화] │
├─────────────────────────────┤
│  처리 상태                   │
│  ○ 전체  ○ 조치 대기  ○ 조치 완료 │
│                             │
│  사고 유형                   │
│  ☐ 전체  ☐ 화재  ☐ 폭발    │
│  ☐ 떨어짐  ☐ 끼임  ☐ 부딪힘│
│  ☐ 붕괴  ☐ 감전  ☐ 중독   │
│  ☐ 온열질환  ☐ 기타        │
│                             │
│  기간                        │
│  [ 시작일 ] ~ [ 종료일 ]    │
│                             │
│  [ 적용하기 ]               │
└─────────────────────────────┘
```

#### 결과 건수
- "전체 N건" 또는 "필터 결과 N건"
- 필터 적용 중: "필터 결과 N건 (전체 M건 중)"

#### 사고 항목 카드
- 사고 유형 아이콘 + 유형명 (좌상단)
- 처리 상태 뱃지 (우상단)
- 등록자 계급 + 성명 (부대 전체 탭에서만 표시)
- 발생 장소
- 등록 일시
- 기한 초과 경고: ⚠ 아이콘 + "기한 초과" 텍스트 (빨강)
- 카드 탭: 사고 상세(MGT-02 기반)로 이동

#### 빈 상태
- "해당하는 사고가 없어요" + 일러스트

---

### 사고 상세 (관리자 버전)

> MGT-02와 동일한 구조, 아래 항목 추가

**추가 버튼:**
- [반려 처리] 버튼 — 조치 완료 상태 건에만 표시
  - 탭 시 반려 사유 입력 바텀시트 표시

**반려 처리 바텀시트:**
```
┌─────────────────────────────┐
│  반려 사유 입력     [반려]  │
├─────────────────────────────┤
│  ┌──────────────────────┐  │
│  │ 조치 내용이 미흡합니다│  │
│  │ ...                  │  │
│  └──────────────────────┘  │
│  (필수 입력, 최대 200자)    │
└─────────────────────────────┘
```
- [반려] 버튼: 사유 입력 시 활성
- 처리 시: 해당 건 비공개 전환 + 간부 푸시 알림 발송 + 완료 토스트

## ADM-03 · 미조치 건 추적 관리

│                             │
└─────────────────────────────┘

---

### 컴포넌트 상세

#### 헤더
- 좌: 뒤로가기 (←)
- 중: "미조치 건 관리"

#### 필터 탭
- [전체 N건] / [기한 초과 N건] (2탭)
- 기한 초과 탭: 빨강 강조

#### 목록 구분
- **기한 초과 구역**: 상단 고정, 빨강 섹션 헤더
- **기한 내 구역**: 기한 초과 구역 아래, 일반 섹션 헤더

#### 사고 항목 카드
- 기본 정보: 사고 유형 / 등록자 / 발생 장소
- 기한 상태 표시:
  - 기한 초과: ⚠ "N일 초과" (빨강)
  - 기한 내: "잔여 N일" (초록/주황 — 3일 이하 주황)
- [촉구 알림] 버튼 — 기한 초과 건에만 표시
  - 탭 시 확인 다이얼로그 표시

#### 촉구 알림 확인 다이얼로그
```
홍길동 하사에게
조치 촉구 알림을 보낼까요?

[취소]  [보내기]
```
- [보내기]: 간부에게 푸시 알림 발송 + "알림을 보냈어요" 토스트
- 이미 당일 발송한 경우: [촉구 알림] 버튼 비활성 + "오늘 발송됨" 표시

#### 카드 탭
- 사고 상세(MGT-02 기반)로 이동

#### 빈 상태
- 전체 탭 빈 상태: "미조치 건이 없어요 👍"
- 기한 초과 탭 빈 상태: "기한 초과 건이 없어요"

#### 헤더
- 좌: 뒤로가기 (←)
- 중: "간부 계정 승인"

#### 필터 탭
- [대기 중 N] / [처리 완료] (2탭)
- 대기 중 탭에 건수 표시

- 성명 (대)

- 카드 전체 탭: ADM-05로 이동

**처리 완료 탭 항목 추가 표시:**
- 처리 결과 뱃지: ✅ 승인 완료 / ❌ 거절
- 처리 일시

- 대기 중 없음: "승인 대기 중인 계정이 없어요"
- 처리 완료 없음: "처리된 내역이 없어요"

#### 헤더
- 좌: 뒤로가기 (←)
- 중: "승인 상세"

#### 가입 신청 정보 카드
| 항목 | 내용 |
|---|---|
| 성명 | 텍스트 |
| 계급 | 텍스트 |
| 군번 | 텍스트 |
| 소속 부대 | 텍스트 (부대명 + 중대) |
| 아이디 | 텍스트 |
| 신청일 | YYYY.MM.DD |

#### 처리 버튼 (대기 중 상태에만 표시)

**[승인] 버튼 — Primary**
- 탭 시 승인 확인 다이얼로그:
```
홍길동 하사의 가입을
승인할까요?

[취소]  [승인하기]
```
- [승인하기]: 계정 승인 처리 + 간부 푸시 알림 발송 + "승인 완료" 토스트 + ADM-04로 이동

**[거절] 버튼 — Secondary (outline)**
- 탭 시 거절 사유 입력 바텀시트:
```
┌─────────────────────────────┐
│  거절 사유 (선택)    [거절] │
├─────────────────────────────┤
│  ┌──────────────────────┐  │
│  │ 사유를 입력해주세요   │  │
│  │ (선택 사항)          │  │
│  └──────────────────────┘  │
│  최대 100자                 │
└─────────────────────────────┘
```
- [거절] 버튼: 사유 입력 여부 무관 항상 활성
- 처리 시: 계정 거절 + 간부 푸시 알림(사유 포함) + "거절 처리 완료" 토스트 + ADM-04로 이동

#### 처리 완료 상태
- 하단 버튼 없음
- 신청 정보 카드 하단에 처리 결과 표시:
```
  ✅ 승인 완료 · 2026.05.16
  (또는)
  ❌ 거절 · 2026.05.16
  사유: 소속 부대 확인 불가
```

## 화면 간 이동 흐름 요약

```
ADM-01 (관리자 홈)
  ├─ 신고 현황 카드 탭 ─────────────── ADM-02 (신고 현황)
  ├─ 미조치 카드 [확인] ────────────── ADM-03 (미조치 추적)
  ├─ 빠른 메뉴 > 사고 등록 ──────────── REG-01
  ├─ 빠른 메뉴 > 신고 현황 ──────────── ADM-02
  ├─ 빠른 메뉴 > 계정 승인 ──────────── ADM-04
  └─ 햄버거 메뉴 ──────────────────── 통계 그래프 화면

ADM-02 (신고 현황 목록)
  └─ 항목 탭 ──────────────────────── 사고 상세 (MGT-02 기반 + 반려 버튼)

ADM-03 (미조치 추적)
  ├─ 항목 탭 ──────────────────────── 사고 상세 (MGT-02 기반)
  └─ [촉구 알림] ───────────────────── 확인 다이얼로그 → 알림 발송

ADM-04 (승인 목록)
  └─ 항목 탭 / [상세] ─────────────── ADM-05 (승인 상세)

ADM-05 (승인 상세)
  ├─ [승인] ────────────────────────── 확인 다이얼로그 → 처리 완료 → ADM-04
  └─ [거절] ────────────────────────── 사유 입력 → 처리 완료 → ADM-04

ADM-06 (보고서 생성)
  └─ [보고서 생성] ─────────────────── 미리보기 → [출력] / [저장]
```

### 컴포넌트 상세

| 컴포넌트      | 내용                                                                   |
|---------------|------------------------------------------------------------------------|
| 보고서 유형   | 월간 / 분기 라디오 선택 — 선택에 따라 기간 선택 UI 변경               |
| 기간 선택     | 월간: 년/월 드롭다운 / 분기: 년 + 분기(1~4분기) 드롭다운              |
| 포함 범위     | 내가 등록한 건만 / 부대 전체 라디오 선택                               |
| [보고서 생성] | 탭 시 로딩 → 미리보기 화면으로 전환                                    |
| [출력]        | 기기 프린터 연결 또는 PDF 출력 (OS 기본 인쇄 대화상자 호출)           |
| [저장]        | 기기 다운로드 폴더에 PDF 저장 + "저장 완료" 토스트                     |

### 상태

| 상태          | 표시                                                        |
|---------------|-------------------------------------------------------------|
| 초기          | 월간 선택, 현재 월 기본값, 부대 전체 선택                   |
| 생성 중       | [보고서 생성] 버튼 로딩 스피너 + 비활성, "보고서를 생성하고 있어요..." |
| 생성 성공     | 미리보기 화면으로 전환                                      |
| 생성 실패     | "보고서 생성에 실패했어요" 토스트 + 생성 화면 유지          |
| 저장 완료     | "파일이 저장되었어요" 토스트                                |
