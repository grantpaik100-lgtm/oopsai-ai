## 2. 테이블 상세 정의

---

### 2-1. `units` — 부대/조직

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| `id` | BIGSERIAL | PK | |
| `name` | VARCHAR(100) | NOT NULL | 부대명 (예: 1사단 1연대 1대대 1중대) |
| `parent_id` | BIGINT | FK → units(id) | 상위 부대 (NULL = 최상위) |
| `level` | VARCHAR(20) | NOT NULL | 부대 계층 (division / regiment / battalion / company 등) |
| `created_at` | TIMESTAMP | NOT NULL DEFAULT now() | |

**설명**
- 자기참조 트리 구조로 부대 계층 표현
- 통계 조회 시 `WITH RECURSIVE`로 예하 부대 전체 집계

### 2-2. `users` — 사용자

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| `id` | BIGSERIAL | PK | |
| `username` | VARCHAR(50) | UNIQUE NOT NULL | 로그인 아이디 |
| `password_hash` | VARCHAR(255) | NOT NULL | bcrypt 해시 |
| `name` | VARCHAR(50) | NOT NULL | 성명 |
| `rank` | VARCHAR(20) | NOT NULL | 계급 (이병~준위 등) |
| `military_id` | VARCHAR(20) | UNIQUE NOT NULL | 군번 (YY-XXXXXX) |
| `unit_id` | BIGINT | FK → units(id) NOT NULL | 소속 부대 |
| `role` | VARCHAR(20) | NOT NULL | 역할 (officer / admin / commander / system_admin) |
| `approval_status` | VARCHAR(20) | NOT NULL DEFAULT 'pending' | 계정 승인 상태 (pending / approved / rejected) |
| `rejection_reason` | TEXT | | 거절 사유 (거절 시에만) |
| `fcm_token` | VARCHAR(255) | | FCM 푸시 토큰 (로그인 시 갱신) |
| `created_at` | TIMESTAMP | NOT NULL DEFAULT now() | |
| `updated_at` | TIMESTAMP | NOT NULL DEFAULT now() | |

**role ENUM 값**
| 값 | 설명 |
|---|---|
| `officer` | 간부 |
| `admin` | 관리자 (인사담당자·안전담당자) |
| `commander` | 지휘관 (대대장급 이상) |
| `system_admin` | 체계관리자 |

**approval_status 흐름**
```
pending → approved (관리자 승인)
pending → rejected (관리자 거절)
```

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| `id` | BIGSERIAL | PK | |
| `reporter_id` | BIGINT | FK → users(id) NOT NULL | 등록 간부 |
| `unit_id` | BIGINT | FK → units(id) NOT NULL | 발생 부대 |
| `occurred_at` | TIMESTAMP | NOT NULL | 식별 일시 |
| `location` | VARCHAR(200) | | 발생 장소 |
| `description` | TEXT | NOT NULL | 식별된 위험 요인 |
| `accident_type` | VARCHAR(20) | NOT NULL | 사고 유형 (아래 ENUM 참조) |
| `risk_level` | VARCHAR(20) | NOT NULL | 위험도 (AI 자동 산출) |
| `ai_recommended` | BOOLEAN | NOT NULL DEFAULT false | AI 추천 유형으로 등록 여부 |
| `action_status` | VARCHAR(20) | NOT NULL DEFAULT 'pending' | 처리 상태 (pending / completed) |
| `action_mode` | VARCHAR(20) | | 조치 방식 (ai_card / manual / null) |
| `action_description` | TEXT | | 조치 결과 텍스트 |
| `action_completed_at` | TIMESTAMP | | 조치 완료 시각 |
| `is_public` | BOOLEAN | NOT NULL DEFAULT false | 공개 여부 (조치 완료 시 true) |
| `review_status` | VARCHAR(20) | NOT NULL DEFAULT 'normal' | 관리자 검토 상태 (normal / rejected) |
| `rejection_reason` | TEXT | | 반려 사유 |
| `embedding` | VECTOR(1536) | | 텍스트 임베딩 벡터 (pgvector) |
| `created_at` | TIMESTAMP | NOT NULL DEFAULT now() | |
| `updated_at` | TIMESTAMP | NOT NULL DEFAULT now() | |

**accident_type ENUM 값**
| 값 | 표시명 |
|---|---|
| `fire` | 화재 |
| `explosion` | 폭발 |
| `fall` | 떨어짐 |
| `entrapment` | 끼임 |
| `collision` | 부딪힘 |
| `collapse` | 붕괴 |
| `electric` | 감전 |
| `poisoning` | 중독 |
| `heat` | 온열질환 |
| `other` | 기타 |

**risk_level ENUM 값**
| 값 | 표시명 |
|---|---|
| `low` | 낮음 |
| `medium` | 보통 |
| `high` | 높음 |
| `very_high` | 매우 높음 |

**is_public 결정 규칙**
```
action_status = 'completed' AND review_status = 'normal' → is_public = true
action_status = 'pending'  → is_public = false
review_status = 'rejected' → is_public = false (반려 시 비공개 전환)
```

## 3. 인덱스 전략

```sql
-- 사고 조회 (내 사고 관리, 신고 현황)
CREATE INDEX idx_accidents_reporter ON accidents(reporter_id);
CREATE INDEX idx_accidents_unit ON accidents(unit_id);
CREATE INDEX idx_accidents_status ON accidents(action_status, is_public);
CREATE INDEX idx_accidents_type ON accidents(accident_type);
CREATE INDEX idx_accidents_created ON accidents(created_at DESC);

-- 유사 사례 검색 (pgvector 코사인 유사도)
CREATE INDEX idx_accidents_embedding ON accidents
    USING ivfflat (embedding vector_cosine_ops)
    WITH (lists = 100);

-- 부대 계층 조회 (재귀 CTE 성능)
CREATE INDEX idx_units_parent ON units(parent_id);

-- 알림 조회
CREATE INDEX idx_notifications_recipient ON notifications(recipient_id, is_read, sent_at DESC);

-- 사고 수정 이력
CREATE INDEX idx_history_accident ON accident_history(accident_id, changed_at DESC);
```

## 4. pgvector 유사 사례 검색

```sql
-- 확장 설치
CREATE EXTENSION IF NOT EXISTS vector;

-- 유사 사례 검색 쿼리 예시 (코사인 유사도 상위 10건)
SELECT
    a.id,
    a.accident_type,
    a.risk_level,
    a.description,
    a.action_description,
    1 - (a.embedding <=> $1::vector) AS similarity
FROM accidents a
WHERE
    a.is_public = true
    AND a.action_status = 'completed'
ORDER BY a.embedding <=> $1::vector
LIMIT 10;
```

> `$1` — 현재 등록 중인 사고의 `description` 텍스트를 `text-embedding-3-small`로 변환한 벡터 (1536차원)
> 임베딩 생성 시점: `POST /accidents` 최종 등록 시 서버에서 OpenAI Embeddings API 호출

## 5. 주요 데이터 흐름 메모

| 시점 | 동작 |
|------|------|
| 사고 등록 완료 (`POST /accidents`) | accidents 행 생성 + embedding 생성 + pgvector 저장 |
| 조치 완료 인증 | `action_status = 'completed'`, `action_completed_at` 기록, `is_public = true` |
| 관리자 반려 | `review_status = 'rejected'`, `is_public = false`, 반려 알림 발송 |
| 사고 수정 | accident_history 행 추가 (변경 필드별 1행) |
| 통계 집계 | `WITH RECURSIVE`로 예하 unit_id 목록 추출 → accidents GROUP BY |
| 미조치 기한 | 서버 스케줄러가 `action_status = 'pending'` 건 기한 계산 → 알림 발송 |
