# 전체 시스템 구조도

### 1.1. 사용자

- **간부:** 아차사고 등록 및 사례 조회 등 일반 사용자 기능 수행
- **지휘관:** 대대장급 이상, 부대 리스크 현황 파악 및 의사 결정
- **관리자 (인사담당자 또는 안전담당자):** 간부 가입 승인 및 신고 현황 운영 관리
- **체계관리자:** 관리자 계정 생성·수정·삭제 및 권한 부여 담당 최상위 운영자

### 1.2. 모바일 앱 (Client)

- 로그인 및 권한 관리
- 권한에 따른 진입 화면 분기 (간부 / 지휘관 / 관리자 / 체계관리자)
- 권한별 바텀 네비게이션 및 메뉴 구성 분리
- 통계 그래프 — 간부·관리자·지휘관 제공 (체계관리자 제외), 권한별 진입 방식 상이
  - **간부:** 홈 화면 하단에서 직접 확인
  - **관리자·지휘관:** 상단 메뉴 버튼(햄버거)으로 진입
  - **통계 범위:** 소속 부대 계층 기준 — 예하부대 사용자는 사단 내 전체, 상위 부대 사용자는 예하 사단들 포함
  - **그래프 구성 (전 권한 동일):**
    - ① 월별/유형별 아차사고 분석 현황 — 범례: 사고 유형 10개 카테고리 (화재·폭발·추락·끼임·부딪힘·붕괴·감전·중독·온열질환·기타)
    - ② 예하부대 아차사고 등록 현황 — 부대별 스택 막대그래프 (조치완료/조치대기), 기간 선택(이번달/3개월/6개월/1년), 카테고리 필터(전체+10개)
- 지휘관 화면은 별도 조작 없이 리스크 평가 → 중점 관리 대상 → 타 부대 사례 → 의사결정 지원 순으로 스크롤 흐름으로 구성
- 아차사고 등록 및 관리
- 사진 촬영 및 텍스트 입력 (시간/상황)
- 위험 요인 체크 및 조치 결과 등록
- 통계 조회 및 푸시 알림 수신

### 1.3. 응용 서버 (Application Server)

- 사용자 인증 및 권한 검증
- 신고 데이터 처리 및 저장
- 부대별 현황 데이터 관리
- 통계 데이터 생성 및 집계
- 알림(Notification) 전송 (미조치 기한 초과 시 관리자 알림 포함)
- AI 분석 서버로의 데이터 전송 및 결과 수신

### 1.4. 분석 기능 (Analysis Engine)

- **사고 유형 분류:** AI가 등록된 사고 내용 및 첨부 사진을 분석하여 카테고리별로 자동 분류
- **위험도 분석:** 사고 등록 마지막 단계에서 AI가 입력 데이터 및 사진을 기반으로 위험도를 자동 산출
- **유사 사례 제공:** 간부가 조치 결과 및 조치 사례 작성 시 참고할 수 있도록 유사 사례 전체 목록을 제공하고, 간부가 직접 확인 후 선택하여 작성에 활용 (수학적/통계적 로직 기반)
- **예방 대책 제안:** AI가 사진 분석 결과를 바탕으로 예방 대책을 텍스트로 제안하며, 예방 대책 적용 시 예상되는 결과를 이미지로 생성하여 함께 제공 (작성 보조용, 최종 내용은 간부가 직접 판단하여 작성)
- **반복 위험 패턴 탐지:** 기준 시점 대비 동일 카테고리의 사고 등록 빈도를 분석하여 반복 위험 현황을 제공 (서버 통계 처리 기반)

### 1.5. 데이터베이스 (DB)

- 사용자 및 부대/조직 정보
- 아차사고 신고 이력 (메타데이터 포함)
- 사진 및 첨부 자료 — 등록 사진 / AI 생성 사진 / 조치 완료 사진 구분 저장 (오브젝트 스토리지 링크)
- 예방 대책 및 조치 결과 내역
- 사고 처리 상태 (조치 대기 / 조치 완료)
- 집계된 통계 데이터

### 1.6. 관리자 대시보드 (Admin)

- **대상:** 인사담당자 또는 안전담당자
- 신고 현황 실시간 모니터링
- 위험도 순위 및 핫스팟 확인
- 자주 찾는 서비스 및 통계 확인
- 미조치 건 추적 및 확인 (조치 기한 초과 시 알림)
- 간부 사용자 가입 승인 (인증 코드 기반)

### 1.7. 체계관리자 대시보드 (System Admin)

- **대상:** 체계관리자
- 관리자(인사담당자·안전담당자) 계정 생성·수정·삭제
- 관리자 권한 부여 및 회수
- 관리자 계정 현황 조회

## 2. 기능 구조도 (Functional Breakdown)

### 2.1. 사용자 영역 (간부)

- **사고 등록:** 아차사고 내용 등록 (음성 인식 및 텍스트 입력 지원)
- **사진 첨부:** 디바이스 카메라를 직접 실행하여 현장 사진 촬영 후 첨부, 첨부 즉시 백그라운드에서 AI 분석 자동 시작
- **상황 입력:** 식별 일시, 발생 장소 등 메타데이터 입력
- **유형 분류:** AI 분석 결과를 참고하여 위험도 및 사고 유형 최종 선택 (사용자 1차 판단, 수정 가능)
- **조치 입력:** 위험 요인 및 초기 조치/예방 대책 입력, AI 제안 내용 및 예상 결과 이미지를 참고하여 작성 가능
- **처리 상태 관리:** 즉시 조치 불가 시 '조치 대기'로 등록 후, 조치 완료 시 완료 사진 추가 업로드 및 '조치 완료' 처리
- **사례 조회:** 등록 건과 관련된 유사 사례 전체 목록 조회 후 간부가 직접 확인·선택하여 작성에 활용 (통계적/수학적 로직 기반, AI 아님)

### 2.2. 관리자 영역 (인사담당자·안전담당자)

- **현황 파악:** 전체 신고 접수 현황 모니터링
- **통계 분석:** 부대별 / 사고 유형별 통계 데이터 시각화
- **위험 식별:** 반복 발생 항목 및 주요 위험 요인 식별
- **추적 관리:** 미조치 건에 대한 조치 상태 추적 및 관리, 기한 초과 시 알림 발송
- **보고서:** 월간/분기별 분석 보고서 자동 생성 및 출력
- **계정 관리:** 간부 사용자 계정 가입 승인 및 권한 관리
- **게시판 관리:** 공지사항 및 안전 교육 자료 업로드/관리

### 2.3. 체계관리자 영역 (System Admin)

- **계정 생성:** 관리자(인사담당자·안전담당자) 계정 신규 생성
- **계정 수정·삭제:** 관리자 계정 정보 수정 및 삭제
- **권한 관리:** 관리자 권한 부여 및 회수
- **계정 현황 조회:** 등록된 관리자 계정 전체 목록 및 상태 확인

### 2.4. 지휘관 영역 (대대장급 이상)

- **리스크 평가:** 등록된 데이터 기반의 통계·알고리즘 처리를 통한 부대별/유형별 고위험 부대 식별 (AI 아님)
- **중점 관리 대상 확인:** 기준 기간(월간 또는 분기) 내 신규 등록 건 중 특정 카테고리(예: 교통 관련)의 사고가 집중될 경우 해당 현황이 두드러지게 표시되도록 서버에서 처리
- **정보 교류:** 타 부대의 아차사고 주요 사례 열람
- **사례 조회:** 간부와 동일한 유사 사례 조회 화면 사용 — 사례 검색 및 필터링 가능
- **예방 대책 의사 결정 지원:** 현 개발 단계에서는 서버 로직 기반으로 데이터 처리 및 제공, 후반기에 AI 분석 접목 예정
